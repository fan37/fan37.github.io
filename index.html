발로 쓰는 개발 로그

Hexo와 Github로 블로그 시작하기

드디어 블로그 플랫폼을 하나 정해 정착하기로 한다(사실 돌아다닌 곳도 없지만).
내가 쓴 글들을 최대한 단순하게 관리할 수 있고 직접 가지고 있을 수 있는 방식으로 정적 HTML 생성기를 택했다.

Github에서는 Github Page라는 무료 호스팅 서비스를 제공한다. 여기에다가 Github에서 제공하는 페이지 제작 도구를 이용해 문서를 작성할 수도 있고, 직접 HTML 파일을 올려서 활용 할 수도 있다. 다만 JSP, PHP 등의 서버사이드 언어는 구동시키지 못한다. 그래서 여기에 올릴 HTML은 마크업 언어로 작성한 파일을 Hexo나 Jekyll 등의 정적 HTML 생성기를 사용해 미리 작성한 후 git을 이용해 업로드 하는 방식을 사용한다.

현재 가장 많이 쓰이는 것은 Jekyll로 보이고, Hexo도 나름 많은 사용자를 얻어가고 있다. 내가 Hexo를 선택한 것은 Node.js로 만들어져 있다는 이유인데, 요즘에 나름 Node.js를 조금 익혀보겠다고 건드려 본 터라 익숙하게 느껴졌고, 혹시나(?) 필요하다면 직접 오류를 수정하거나 기능을 추가할 수 있다는 점도 고려했다(Jekyll은 Ruby 기반).

일단 해보기 - Hexo 설치

Hexo를 설치하고 새로운 포스트를 생성하여 확인하는 과정까지가 매우 손쉽기 때문에, 일단 바로 해보도록 하자. Hexo 사이트에도 잘 설명되어 있지만, 여기서 더욱 간단하고 빠르게 설명해본다. 감을 잡기 위한 기본 사용법만 소개할테니 자세한 내용은 Hexo 사이트를 참조하자.

우선 Hexo를 사용하기 위해서는 Node.js를 설치해야 한다. 설치되어 있지 않다면 nodejs.org에서 다운로드 받아 설치한다.

Node.js가 설치되면 커맨드 라인에서 패키지 관리자 npm을 사용해 아래와 같이 Hexo를 설치한다.

$ npm install hexo-cli -g

Hexo가 설치되면 원하는 이름으로 블로그를 생성하고 해당 디렉토리에서 npm install을 실행하여 필요한 패키지가 설치되도록 한다.

$ hexo init myblog
$ cd myblog
$ npm install

블로그가 생성되었다. 아래 명령으로 웹서버를 구동해 생성된 블로그를 확인해볼 수 있다.

$ hexo server

http://localhost:4000 주소로 접속해서 확인!

hello world라는 하나의 포스트가 생긴 것을 확인할 수 있다. 이 포스트는 블로그 디렉토리의 source/_posts 밑에 .md 파일(마크다운 파일 확장자)로 존재한다. 이 문서를 텍스트 에디터나 마크다운 편집기로 수정하여 포스트 내용을 수정할 수 있다.

새로운 포스트를 추가하려면 다음과 같이 하면 된다.

$ hexo new "포스트 이름"

정적 파일을 생성해 실제로 Github에 올려보기

다음 명령을 통해 마크다운으로 작성된 포스트를 실제 템플릿을 통해 HTML로 생성한다.

$ hexo generate

명령 실행 후에 public 디렉토리에 실제 HTML과 관련 리소스가 생성된다. 이를 Github에 올리면 그대로 멋진 블로그가 된다. 이것을 자동으로 해주는 npm 패키지가 있다.

블로그 디렉토리에 다음과 같이 hexo-deployer-git을 설치한다.

npm install --save hexo-deployer-git

그리고는 다음 명령어 하나면 public 디렉토리에 생성된 HTML들을 Github에 올려준다.
참~ 쉽죠?
